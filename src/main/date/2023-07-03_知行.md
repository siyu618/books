
### 1. 管理入口的彷徨

1. 技术管理者的技术评估
	1. 技术项目的结果评估
		1. 这究竟是个什么事情
		2. 希望得到什么结果
		3. 要从哪些维度去衡量结果？从哪些技术指标去验收成果？
	2. 技术可行度评估
		1. 能过能做： 有没有能力做到， 是能力问题。
		2. 值不值得做：指能力允许，但是否有足够的收益值得做，这个是选择问题。 
			1. 投入的资源成本
			2. 技术维护的成本
			3. 机会成本
			4. 协作成本
	3. 技术风险评估
		1. 建立技术学习机制
		2. 专项技术调研项目化
		3. 和技术专家交流
		4. 听取工作汇报
2.  管理者和团队协作关系
	1. 发号施令型： 
		1. 适用于强执行的场景。
	2. 以身作则型：
	3. 激发辅导型
	4. 无为而治型
3. 领导力风格
	1. 指令式管理：重事不重人，关注目标和结果，喜欢发号施令而不会亲历亲为。 
	2. 支持式管理：重人不重事，习惯带头冲锋亲历亲为，特别在意团队成员的感受，并乐于分担他们的工作。 
	3. 教练式管理：重人也重事，关注全局和方向，并在做事上基于教练式辅导和启发。
	4. 授权式管理：不重人也不重事，关注目标和结果，不关心过程和人员发展。 
4. 能力三核模型
	1. 知识：知道和理解的内容和信息。
	2. 技能：操作并完成的技术。
		1. 这个层次的能力有一定的可迁移性。
		2. 比如：快速学习的能力，进度控制的能力。
	3. 才干：思维、行动和感受模式以及品格特质。
		1. 这个层次的能力可迁移性是最强的。





### 2. 管理的基本框架
1. 管理的三要素
	1. 带人
	2. 目标
	3. 做事
2. 领导力的定义：领导力就是动员大家为了共同的愿景努力奋斗的艺术。
3. 管理三明治
	1. 角色认知
	2. 管理规划： 看方向
		1. 四要素
			1. 职能：你团队是干什么的
			2. 目标：你要产出什么
			3. 团队：你要依靠谁
			4. 路径：你要投入什么
		 2. 目标拆解为路径，其核心目的是为了资源预算。确保规划的可行性。 
		 3. 目标拆解为计划，其核心目的是为了落地执行。确保执行过程可控。
	3. 团队建设：带人
		1. 团队建设六要素
		2. 一个目标：团队建设：战斗力
			1. 个体：动力
				1. 员工实力：能力培养
				2. 使用实力的意愿：员工激励
			2. 个体间：凝聚力
				1. 排兵布阵：团队分工
				2. 步调节奏：协作默契
			3. 团队：耐力
				1. 新老强弱：梯队建设
				2. 归属认同：团队文化
	4. 任务管理：做事情
		1. 任务管理三要素
			1. 事前：做什么，排优先级，轻重缓急。
			2. 事中：怎么做，执行落地，有效执行。
			3. 时候：怎么做更好，自动驾驶，流程机制。
	5. 管理沟通
		1. 管理沟通的方方面面
			1. 沟通场景：向上沟通，向下沟通，横向沟通。。。
			2. 沟通目的：建立关系，同步信息，表达情感，说服影响
			3. 沟通内容：计划，进展，风险，协同，人员
			4. 沟通方式：当面，电话，消息，电子邮件，协作平台
			5. 沟通工具：3F倾听，积极主动式反馈，回放确认
			6. 沟通意识：主动同步、换位思考、闭环意识、结论优先

### 3. 角色认知
1. 角色认知的必要性 NLP逻辑层次
	1. 环境：何时何地（外部资源）
	2. 行为：做什么（行动安排）
	3. 能力：怎么做（能力技巧）
	4. 信念和价值观：为什么（信念价值）
	5. 身份：我是谁（身份角色）
	6. 精神：为了谁（使命意义）
2. 角色的力量
	1. 部门总负责人：追求的是事情的可能性
	2. 中间管理者：追求的是事情的性价比
	3. 工程师：追求的是事情的确定性。
3. 角色变化

|维度|工程师（个人贡献者）|管理者（团队领导）|
|---|---|---|
|工作职责|拉车 - 完成本职工作|驾车 - 引领整个团队前进|
|负责对象|对自己负责|对公司（上级）和团队（下级）都负责|
|关注焦点|脚下的路 - 过程导向|远方的目标 - 结果导向|
|能力要求|专业单一|多为复杂|
|任务来源|直接上级|主动规划|
|实施手段|主要靠自己|主要靠团队|
|合作维度|平级合作为主|多维合作维度|
|合作关系|平级竞争关系|全面合作关系|
|思维方式|确定性思维|可能性思维|
|技术视角|技术实施视角|技术评估视角|



4. 管理误区
	1. 管理更多的的是一门实践科学，从知道到做到，还需要刻意练习。
	2. 六大误区
		1. 过程导向，被动执行
		2. 大包大揽，个人英雄主义
		3. 带头大哥、当家保姆
		4. 单一视角、思维固化
		5. 自扫门前雪、固守边界
		6. 身在曹营心在汉，患得患失

### 4. 管理规划
1. 管理规划四要素
	1. 职能：团队是干什么的
	2. 目标：团队要产出什么
	3. 团队：团队是什么状态，以及未来是什么状态
	4. 路径：所需要投入的资源以及理由
2. 职能
	1. 三个问题
		1. 公司为什么要设置这个团队，对这个团队的期待是什么
		2. 这个团队如果要持续存在下去，其独特价值是什么
		3. 用什么维度来衡量团队价值的高低
	2. 两个层次
		1. 职责：团队职能的下线。解决团队生存问题。
		2. 升华：团队职能的上线。解决团队实现问题。 
	3. 团队职能的设定
		1. 收集信息
			1. 向上沟通
			2. 向下沟通
			3. 左看右看
			4. 你的理解
		2. 提炼和生化
			1. 职责的提炼
			2. 使命的升华
				1. 使用基于“结果“的描述
			3. 确定衡量维度
				1. 服务端团队使用 稳定性、性能、可维护性等维度
				2. 前端团队注重 开发效率、兼容性、规范性等维度
				3. 数据团队，关注准确性、及时性、安全性等维度
		3. 确认和主张
			1. 和上级确认，得到上级的认可和支持
			2. 向平级和下级主张。
3. 目标
	1. 方面：意义、原则、维度、形式、挑战
	2. 意义
		1. 目标意味着期待
		2. 意味着资源的有效配置
		3. 意味着执行力
		4. 意味着凝聚力
		5. 意味着激励
	3. 原则
		1. SMART 原则
	4. 目标设立的维度
		1. 业务目标：业绩产出
		2. 团建目标：团队发展
		3. 专业目标：专业能力
	5. 目标设定的形式
		1. 可量化的指标 KPI
		2. 不可量化的指标，用关键结果来衡量 OKR
		3. KPI 和 OKR 的区别
			1. KPI 的逻辑是：无论如何都要确保KPI的达成，追求的是结果达成的确定性。 
				1. 更适用于规则成熟、最求执行性的组织。
			2. OKR的逻辑：对 O 有帮助的都可以做，注重的是目标达成的可能性。
				1. 更适用于开放性强，追求创造性的组织。 
	6. 目标设定的挑战
		1. 基于现有资源做目标，而不是基于远方的目标往前推
		2. 目标不明确
		3. 目标设定好之后，不向下传达
		4. 苦恼于目标的频繁变更
4. 团队
	1. 三个视角
		1. 团建目标的设定
		2. 资源角度看团队的构成
		3. 人才发展角度来规划梯队
	2. 团队规划之目标视角
		1. 团队规模
		2. 团队的分工
		3. 团队的梯队
	3. 团队规划之资源视角
		1. 在 HR 眼里，每个团队就是一边资源，只不过这笔资源是以”人力“来体现的。 在公司财务眼里，每个团队都是一笔成本，只不过这笔成本是以人力来体现的，叫人力成本。
		2. 从资源视角来看待团队，是一个成熟管理者的标志之一。 
		3. 怎么才能够合理推算？ 
			1. 首先， 是你对业务的理解，以及你希望达成的目标。 
			2. 其次可以参照行业资源配比情况。 
	4. 团队规划之呈现
		1. 组织结构图
		2. 资源盘点情况
		3. 重点培养对象
5. 路径
	1. 上级一般都是资源敏感型的。 
	2. 在使用”增加人手“这件事情之前，考虑三个问题
		1. 资源类型的丰富性
			1. 团建费用、培训费用、差旅费用、办公费用
			2. 时间资源
			3. 信息资源
			4. 权限资源
		2. 手段选择的多向性
			1. 站在全局来看，90 分的方案未必就比 70 分的方案好。
		3. 人才招聘的必要性
			1. 招聘作为一种成本高、见效慢的解决问题的手段，核心是看长线是否需要。 
	3. 结果评估的三要素
		1. 进度：从完成时间来看，结果是否满足预期
		2. 质量：是否保证了项目质量
		3. 效果：是否实现了期待的功能，达到了期待的结果。
6. 管理规划报告
	1. 意愿
	2. 时机
	3. 形式
	4. 沟通

### 5. 团队建设
1. 个体
	1. 能力
		1. 定义你所谓的员工能力模型
		2. 提升员工能力的学习方法
		3. 激发员工的学习动力
	2. 激励
		1. 激励的最大挑战是激励手段的匮乏
		2. 激励经历了三个阶段的额发展，只是经理时代的激励主要考虑驱动力3.0：自主投入即内驱；通过自主、专精和目的可以有效提升内驱力。
		3. 员工幸福感 - PERMA模型：积极情绪，人及关系，投入，成就，意义。
		4. 激励方案要点表。
2. 个体之间
	1. 分工
		1. 分工的目的是：为了做更大更复杂的事情。
		2. 常见的组织结构和分工。 
		3. 常见的分工问题。 
		4. 虚拟组织介绍。 
	2. 协作
		1. 整体统筹。
		2. 建立机制。 
		3. 提升互信。
		4. 不断练习。
3. 整体
	1. 梯队
		1. 选才：人才和团建的一致性； 人才和管理者的互补性。
		2. 育才：对齐彼此期待；做好工作授权；建立反馈机制。
	2. 文化
		1. 命名它：提炼团队文化
		2. 主张它：宣贯团队文化
		3. 追求它：践行团队文化。
		4. 团建活动：设计一次团建活动考虑的四个问题： 初衷、问角色、问目的、问手段。 
		5. 团队凝聚力：凝聚力的提升要从共同愿景、提升归属、增进了解和面对挑战四个角度着手。

	2. 职责明确
	3. 机制健全
	4. 沟通到位
3. 事后：流程机制
	1. 简单原则
	2. 关键原则
	3. 问责原则
	4. 实用原则

### 7. 管理沟通
1. 管理沟通的两个视角
	1. 管理视角： 这是一个什么管理问题。是角色认知问题，还是管理规划、团队建设、任务管理问题？具体在哪个要素上 - 是目标问题、激励问题，还是优先级问题。 
	2. 沟通视角： 对于这个管理问题，沟通该怎么做。沟通的目的是什么？ 要准备哪些内容？以何种方式来沟通？
2. 管理沟通的挑战
	1. 管理沟通最具挑战的问题：向上沟通、员工激励和团队凝聚力的提升。 
	2. 不同类工作的特点：
		1. 技术工作：操作机器、学习语言、设计算法、开发功能、遵循规范
			1. 主要和客观事物、自然规律打交道。
			2. 确定性、精确性和稳定性。 
		2. 项目管理：需求明确、制定计划、把控流程、推动执行、通报进展
			1. 核心还是做事，还是基于规则和规范的。 
		3. 和下级合作：分配任务、跟进进展、辅导帮助、激发动力、评价结果
			1. 流程机制是用来保障工作的“下限”的。
			2. 而激励是激发团队工作“上限“的。 
		4. 和上级合作：领取任务、领会意图、提供建议、申请资源、寻求指导
			1. 不确定性多，而且确定性很少。 
3. 管理沟通的基本框架
	1. 四个要素：目的、通道、内容、影响力。 
		1. 目的是沟通的核心。
		2. 通道是沟通的载体。 
		3. 内容是沟通的主体。
		4. 影响力是沟通的有效力量。
	2. 沟通目的
		1. 建立通道： 建立沟通关系和沟通渠道，也就是你要和谁建立什么样的沟通通道，以什么方式和频度进行沟通。
		2. 同步信息：把相互不了解的信息同步给对方。
		3. 表达情感：有时候，沟通知识为了表达某种情感，例如表达焦虑和压力。
		4. 输出影响：再工作中，这类目的的沟通非常多的， 例如提出建议希望对方采纳，管理上级的预期、和员工沟通绩效、向上申请资源。 
	3. 沟通通道
		1. 建立良好的沟通通道
			1. 稳定性：这个通道是稳定可靠的，不会动不动就谈崩或断了联系，及时有点误会双方也能够相互包容和谅解。
			2. 性能：沟通的效率，几这个沟通通道的效果和成本之比。
		2. 如何建立沟通通道
			1. 从下面四个方面着手：沟通意愿、事务特点、沟通风格和信任关系。
			2. 沟通意愿：
				1. 沟通的基本前提。
			3. 评估事务特点
				1. 是否重要、紧急、敏感、正式、例行。
			4. 了解沟通风格
				1. 根据人来选择沟通的方式。
				2. 对读者型：应该以书面沟通为主，口头共同为辅。 
				3. 对听着型：应该以口头沟通为主，书面沟通为辅。
			5. 审视信任关系
				1. 查看是否可以简化沟通。 
	4. 沟通内容
		1. 内容选取：沟通什么、不沟通什么。
		2. 呈现逻辑
			1. 原则：标题清晰、指向明确、结论有限、结构 MECE
		3. 倾听。
		4. 确认。回放式问题。
	5. 影响力
		1. 职权影响力
		2. 非职权影响力
	6. 管理沟通的四项积累
		1. 稳定：积累的维度应该符合以下要求
			1. 特别有效，且持续有效。
				1. 对管理沟通来说很关键，积累一些效果不明显的能力是不值得的。 
			2. 可持续积累。
				1. 随着积累的增多，对管理沟通的帮助越来越大。
		2. 四项积累
			1. 管理逻辑
			2. 通道品质
			3. 工具流程
			4. 影响力
4. 两个重要的沟通技术
	1. 通道品质好就靠通道，通道品质不好就靠沟通技术。
	2. 倾听
		1. 3F 工具： Fact、Feel、Focus
			1. 事实
				1. 列举事实信息
				2. 确认对方的客观情况
			2. 感受：
				1. 对方对事实的感受和态度是什么
				2. 对方的感受和态度是否一致
			3. 意图
				1. 对方的焦点在哪里，我的焦点在哪里
				2. 对方的意图是什么，我的意图是什么
			4. 共识
				1. 是否达成共识，并确认共识
				2. 接下来要怎么做
	3. 发问：也就是提问
		1. 封闭式问题
			1. 要降低回答的成本
			2. 要收敛和聚焦话题的时候
			3. 要得出结论并加以确认的时候
		2. 开放式问题
			1. 手机信息和观点的时候
			2. 激发对方表达意愿的时候
			3. 缓和谈话气氛的时候
5. 影响力
	1. 职权影响力
		1. 传统意识
		2. 职位因素
		3. 资历因素
	2. 非职权影响力
		1. 六方面：互惠、承诺一致、社会认同、喜好、权威和稀缺
		2. 四维八因素
			1. 信任（人）
				1. 品格：了解程度、人品认同度
				2. 历史表现：承诺一致度、及时响应度

	2. 向下沟通
		1. 四类问题
			1. 关于如何批评员工

					1. 人没问题原则：对事不对人。批评事情，不要打击人，更不能给人贴标签。
					2. 具体性原则：指出具体哪里做的不好，让对方容易认同。
					3. 面向未来原则：体现负面的暂时性和过去性，并提供改变的出口。
				2. BID 发展性反馈
					1. Behavior 行为：指出需要改善的具体行为
					2. Impact 影响：指出该问题带来的影响
					3. Desired behavior 期待的行为：指出对未来行为的期待

						2. 表达欣赏

						4. 持续挖掘
				2. 对总聊不到一个频道的员工
					1. 沟通层次图：从事实、感受、意图三个层面来对齐。
				3. 对于捉摸不透的员工
					1. 沟通层次图：从事实、感受、意图三个层面来对齐。

				2. 认同高工的价值：授权，尤其是技术问题的授权，给他们充分的发挥空间。 
				3. 支持和帮助：帮助工程师解决他们不擅长的问题，一方面保证了成功，另一方面还积累的影响力。

	3. 横向沟通
		1. 也叫跨团队沟通，是指和没有直接汇报关系的合作方之间的沟通，者主要是相对于有直接上下级关系的合作方之间的沟通。

			1. 意识到哪些信息需要同步，并主动同步
			2. 和对方约定好沟通机制，并监督机制的有效运转
		3. 对于有明显共同上级的合作方，双方合作的底层逻辑是组织协同。 
		4. 对于无明显共同上级的合作方，双方合作的底层逻辑是利益互换。
7. 绩效沟通
	1. 绩效沟通的挑战
		1. 对绩效制度的不认同
		2. 绩效管理的各种困惑
		3. 沟通过程的各种挑战
	2. 绩效制度的本质
		1. 组织意义：从人的角度来看，对于组织来说，绩效制度就是一个”人才筛选器“，或者称之为”组织优化器“。
		2. 业绩意义：从事的角度来看，对于实现组织目标来说，绩效制度明确了公司内部各个团队和个人的努力方向，是工作的”指南针“。
		3. 结论
			1. 绩效制度设计的初衷是为了达成业绩，但在当前时代的背景下，其核心价值在于公司内部各个组织间的协同，而不能确保业绩目标的实现。 
				1. **作为管理者，一定要重视绩效管理**
				2. **作为负责绩效的 HR 和管理者，绩效制度设计的越简洁越好** 
			2. 绩效制度对于公司来说，本质就是一个优化组织的”筛选器“， 这是绩效制度的 **”第一性原理“**。 
	3. 绩效制度的窘境
		1. 制度的两个要素：普适性、稳定性。
	4. 绩效管理
		1. 绩效对于管理者和员工来说，归根结底是一个上下级之间的 **“工作协议”**， 这个协议约定了上级将来以上那么方式考核夏季，并以此作为依据来评价下级的工作做得好还是不好。 
		2. 五个步骤
			1. 绩效协定：管理者希望接下来员工把工作重点放在哪里，管理者需要现有一个整体的绩效计划的。 
			2. 绩效确认：管理者和员工狗咬认同这个计划或方案。这是很重要的一步。管理者单方面认同的绩效计划，不能作为有效的“工作协议”。因此有些公司的绩效是需要员工签字的。 
			3. 绩效维护：如果员工工作内容有重大调整，绩效计划就要及时更新。既然是“工作协议”，只要双方认同就可以调整，并没有不能改的道理。
				1. 管理者应该是绩效更新的主动发起者，否则绩效评估和沟通时被动的会是管理者。 
				2. 如果绩效考评约定的是“刻度”和具体的工作内容，会很容易引起变更，如果约定的是维度，会更加稳定。
			4. 绩效评估：管理者需要对员工在改绩效周期内的工作表现进行认真的评估并打分。
				1. 评估不能基于自己的印象和感受，需要基于员工的实施表现和绩效计划的约定。
				2. 员工自评的各项分值高低分布比分值本身更加有参考意义。但是无论如何，管理者最终要忠于自己的判断，不能被员工的自评带偏。 
				3. 绩效评估是绩效沟通的前提和基础，要在评估的时候，厘清员工业绩表现要点和后续的发展路线。 
			5. 绩效沟通：就绩效评估结果和员工进行充分沟通，达到对齐信息，辅导工作和激励员工的效果。 
	5. 绩效沟通
		1. 绩效沟通的目的
			1. 信息对齐
				1. 事实信息 （fact）：降低信息的盲区
				2. 判断逻辑（feel & judgement）交换评价标准和对事实的判断。
				3. 双方的意图（focus）：了解对方的期待。 
			2. 工作辅导
				1. 改进建议：对过去做的做法提出优化建议。比如“考虑可扩展性” “及时求助” “主动反馈” 等。 
				2. 努力方向：对未来的做法提出期待。比如“共同指定一个方案来提升员工某项能力”。
				3. 职场法则：扩展员工对职场的理解和认识。 比如“职场本质是价值兑换” “职场是马拉松” 等。 
			3. 员工激励
				1. 坦诚而深入的沟通本身就是激励。 
				2. 上级的理解、上级的重视、上级的支持和不放弃。 
				3. 上级的期待和指引。
		2. 绩效沟通的步骤
			1. 搞定自己
				1. 诚意：审视自己是否表里如一，真诚而不自欺。
				2. 正心：审视自己的角色，让角色给你力量。 
				3. 理直：审视自己的基本逻辑，理直才能气壮。
			2. 搞定绩效
				1. 审视协议：提前回顾绩效计划，如果发现有不合理的地方还有时间去更新和调整。
				2. 搜集事实：提前review员工表现，手机反馈和具体案例，让我们的评价尽可能客观和全面。 
				3. 提前计划：提前思考员工的发展和后续安排，这样才可以在绩效沟通过程中有备而来。 
			3. 搞定沟通

9. 常见的沟通误区
	1. 视角问题：沟通仅从自己出发，对管理者的角色和视角认知不够。 
	2. 姿态问题：总在防卫，准备随时战斗。 
	3. 方式问题：先给人贴标签，对人不对事。 
	4. 意识问题：沟通没有形成闭环 
	5. 初衷问题：只给抱怨不给建议。
